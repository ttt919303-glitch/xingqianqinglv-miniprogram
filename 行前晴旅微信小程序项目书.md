# 行前晴旅微信小程序项目书

## 一、项目简介

“行前晴旅”是一款面向旅行出行场景的微信小程序，副标题为“轻松规划每一次出发”。作品定位为可爱简洁风格的旅行行程助手，帮助用户把地点、交通、时间、备忘、账单和行李清单整理在一个旅行计划里，既能服务出发前准备，也能服务旅行中的路线调整和花费记录。

本作品主题积极健康，内容聚焦旅行规划与生活效率提升。界面采用蓝天、白色卡片、暖色插画感元素组成清新明亮的视觉风格，适合课程展示和日常使用场景。

## 二、需求分析

### 1. 用户需求

旅行前用户通常需要同时确认目的地、日期、交通方式、游玩景点和行李物品。若信息分散在聊天记录、备忘录和购票软件中，容易出现遗漏。因此，小程序需要提供一个轻量、直观、便于浏览的行前整理工具。

### 2. 功能需求

- 首页展示即将出发的行程、倒计时、准备进度和旅行小贴士。
- 物品清单按类别展示，包含证件、电子产品、衣物、旅行用品、洗漱用品、药品等，并提供推荐物品库帮助用户快速补充清单。
- 分类详情页支持查看全部、未打包、已打包物品，并能勾选准备状态。
- 旅行详情页聚合行程路线、交通卡片、旅行备忘、账单预算、地点收藏和行李清单入口，模块卡片可直接跳转，旅行备忘支持新增、编辑、完成、删除和关联景点/交通，形成单个旅行计划的管理中心。
- 行程计划页展示出行日期、交通方式、交通时间、景点安排，并提供“时间优先、少换乘、距离优先”三种路线推荐策略。当前版本已支持景点上移/下移排序、每段交通方式选择和自动重算时间轴，并接入腾讯位置服务 WebService，可优先获取真实路线，接口不可用时保留本地推荐方案。
- 账单页支持记录、编辑、删除预算和实际花费，统计总预算、已花费、剩余金额以及交通、住宿、餐饮、门票等分类金额，并在超预算时给出提示。
- 地点收藏夹作为旅行素材库，支持筛选、新增、编辑、删除地点，并可将收藏地点一键加入当前行程。
- 模板页提供商务出行、周末短途、毕业旅行、亲子旅行等不同场景参考。
- 我的页面展示作品简介、准备统计和后续扩展方向。
- 数据管理支持清空勾选、恢复初始数据、删除自定义行程和删除自定义物品。

### 3. 非功能需求

- 页面结构清晰，页面数量控制在 4 至 10 页之间。
- 页面深度控制在 2 至 3 层，方便用户快速返回和切换。
- 文字简洁通顺，信息表达明确。
- 设计风格协调统一，色彩搭配清新自然。
- 项目文件命名规范，能在微信开发者工具中正常运行。

## 三、系统设计

### 1. 页面结构

小程序共设计 10 个页面：

- 首页：展示行程卡片、倒计时、准备进度和小贴士。
- 旅行详情页：聚合行程路线、交通卡片、旅行备忘、账单预算、地点收藏和行李清单入口。
- 物品清单页：展示各类物品的准备进度，并提供旅行场景推荐物品库。
- 分类详情页：展示单个分类下的物品并支持勾选。
- 行程计划页：展示交通安排、当天景点、推荐路线，并支持景点顺序编辑、分段交通方式选择和时间/预算重算。
- 账单页：记录、编辑、删除预算和实际花费，显示分类统计和超支提醒。
- 地点收藏夹页：展示旅行素材库，支持筛选、新增、编辑、删除地点，并支持一键加入行程。
- 模板页：展示不同旅行场景的准备模板。
- 我的页面：展示项目说明和统计信息。
- 新增页面：提供新增行程或物品的演示入口。

### 2. 导航结构

底部导航包含：首页、路线、清单、账单、我的。模板和新增页面作为低频入口放在首页、详情页和相关页面中进入，使高频的路线、清单、账单更容易触达，同时页面深度保持在 2 至 3 层之间。

### 3. 数据结构

项目采用本地静态数据作为基础数据，包括行程数据、清单分类数据、推荐物品库、模板数据、结构化交通卡片、收藏地点、旅行备忘和账单数据。用户新增的行程、物品、打包状态、路线手动顺序、分段交通方式、账单记录、交通卡、备忘和收藏地点使用 `wx.setStorageSync` 保存，便于用户切换页面后保持操作结果。系统还提供恢复初始数据功能，方便课堂演示时重新回到默认状态。

## 四、功能实现

### 1. 首页功能

首页展示上海、杭州、苏州三个示例行程。每张行程卡片包含目的地、日期、出发倒计时、准备数量和行程小贴士。用户点击行程卡片可以进入对应的行程计划页。

### 2. 物品清单功能

物品清单页按分类展示准备情况，每个分类显示图标、名称、已准备数量和进度条。用户可以进入分类详情页查看具体物品，也可以使用“一键打包完成”快速完成演示。页面新增推荐物品库，包含短途旅行、海边旅行、亲子旅行、商务出行、露营旅行等场景，用户点选后即可把相关物品加入自己的清单，并自动避免重复添加同名物品。

### 3. 分类详情功能

分类详情页提供“全部、未打包、已打包”三个筛选选项。物品以卡片网格展示，包含图标、名称、数量和勾选状态。点击物品卡片即可切换是否已准备。

### 4. 行程计划功能

行程计划页展示目的地、日期、交通工具、交通时间、耗时和当天景点安排。系统会根据景点时间、区域、建议停留时间和简化坐标生成本地路线方案，支持“时间优先、少换乘、距离优先”三种策略切换。页面会展示推荐顺序、预计总用时、交通衔接时间、非步行换乘次数、交通预算和每段交通建议。例如上海行程可根据策略生成“外滩 -> 南京路步行街 -> 豫园 -> 陆家嘴”等路线说明，帮助用户减少路线折返。

在此基础上，路线页新增编辑模式。每个景点以时间轴卡片展示，包含时间段、景点名称、备注、建议停留时间和下一段交通方式。用户可以通过上移、下移按钮调整景点顺序，也可以在两个景点之间选择步行、地铁、打车、骑行等交通方式，系统会重新估算游玩时间、交通时间、预计结束时间和交通预算，并在行程偏满或景点距离较远时显示提醒。地图预览区显示景点序号 marker 和路线连线，点击 marker 后会显示景点名称、区域、推荐时段和备注。

在此基础上，项目接入腾讯位置服务 WebService。用户点击“获取腾讯路线”后，小程序会对景点名称进行地理编码，并按每段路线调用腾讯地图路线规划接口，生成真实距离、预计交通时间和路线来源说明。若接口调用失败、网络不可用或小程序后台未配置合法 request 域名，页面会提示原因并继续展示本地路线推荐，保证核心功能可用。

本机开发演示阶段已在项目配置中关闭开发者工具 URL 校验，便于课堂环境下直接调试接口。若进行真机预览、上传体验版或正式发布，仍需要在微信公众平台后台配置 `https://apis.map.qq.com` 为 request 合法域名。

为避免公开仓库泄露接口凭据，腾讯位置服务 Key 使用本地私有配置文件 `config/tencent-map.js` 保存，仓库中仅提交 `config/tencent-map.example.js` 作为填写示例。

### 5. 旅行详情、账单和收藏夹

旅行详情页是单个旅行计划的总入口，集中展示路线、交通、备忘、账单、收藏夹和清单入口。交通信息以车票卡片呈现，支持新增、编辑、删除去程、返程、市内交通等结构化记录，包含班次号、出发到达时间、座位、票价和提醒文字；编辑预设交通卡时会转为正式交通记录并隐藏原始生成卡，避免重复。旅行备忘支持输入待办内容，选择出发前、路上、入住、游玩、返程等分类，填写日期、提醒时间、关联景点和关联交通，并支持完成勾选、编辑和删除。账单页区分预算和实际花费，按交通、住宿、餐饮、门票、购物、其他进行分类统计，并支持日期、地点、支付方式、备注、编辑、删除和超支提醒。地点收藏夹以明信片风格展示想去地点、推荐停留时间、人均预算和适合时间，支持筛选、新增、编辑、删除，并支持一键加入当前行程。

### 6. 模板与我的页面

模板页提供多种旅行场景，帮助用户根据出行目的快速建立准备思路。点击模板后会向清单中补充常用物品，并避免重复添加同名物品。我的页面展示应用名称、准备进度、行程数量、清单分类数量和作品说明，同时提供清空勾选与恢复初始数据功能，使作品主题更加完整。

## 五、界面设计

小程序整体采用“蓝天旅行 + 暖色插画 + 白色卡片”的视觉方向。主色为清爽蓝，辅色为薄荷绿和浅橙，背景为浅灰白。页面信息以卡片承载，重点内容使用较大的标题和数字展示，提高浏览效率。

首页使用天空渐变、云朵、热气球、飞机等轻量装饰元素营造旅行氛围。清单页和详情页注重实用性，通过进度条、分类图标和筛选标签帮助用户快速判断准备状态。

## 六、测试说明

测试时主要检查以下内容：

- 微信开发者工具能够正常打开并编译项目。
- 底部导航的五个入口均能正常切换。
- 首页三个行程卡片显示正确，点击后能进入行程计划页。
- 物品清单页能正确统计已准备数量和总数量，推荐物品库能一键加入物品并避免重复。
- 分类详情页的“全部、未打包、已打包”筛选功能正确。
- 点击物品后勾选状态能够变化，并在返回页面后保持。
- 分类详情页支持本类全选、本类清空，长按自定义物品可删除。
- 旅行详情页模块卡片能进入路线、账单、收藏夹和清单等功能入口。
- 行程计划页能展示交通方式、交通时间、景点安排和推荐路线，切换路线策略后路线摘要和分段建议能同步刷新。
- 路线编辑模式支持景点上移、下移，并能同步刷新时间轴摘要和地图 marker 预览；每段交通方式切换后能重算用时和预算。
- 地图 marker 点击后能显示对应景点信息。
- 交通卡片支持新增、编辑和删除结构化交通信息。
- 旅行备忘支持新增、编辑、完成、删除和关联景点/交通。
- 账单页能新增、编辑、删除预算或实际花费，并能正确统计总预算、已花费、剩余金额、分类金额和超支提醒。
- 地点收藏夹能筛选、新增、编辑、删除地点，并将地点加入当前行程，更新行程景点列表。
- 新增页可以保存自定义行程和自定义物品，保存后能在对应页面显示。
- 行程路线页点击“获取腾讯路线”后，能优先展示腾讯地图路线；接口不可用时能回退到本地推荐。
- 模板页重复套用时不会重复添加同名物品。
- 我的页面可以清空勾选并恢复初始数据。
- 页面文字没有明显错别字，布局层次清楚。

项目还预留了 `agent-bridge` 自动化联调配置，可配合 `miniprogram-agent-bridge` 在微信开发者工具中执行新增物品、新增行程、路线推荐页等冒烟流程，便于后续检查页面跳转、数据保存和页面 data 报告。

## 七、项目总结

本项目围绕旅行出行准备这一明确主题展开，避免了功能堆砌，重点突出“行前规划”和“物品整理”。通过首页行程卡片、分类清单、景点时间轴和模板场景，小程序形成了较完整的使用闭环。

当前版本主要采用本地数据实现，并接入腾讯位置服务作为增强能力，适合课程作业展示和基础交互演示。项目已经具备行程展示、旅行详情、物品管理、推荐物品库、模板套用、本地路线推荐、分段交通选择、路线顺序编辑、地图 marker 预览与点位提示、腾讯地图路线获取、结构化交通卡片、正式旅行备忘、账单编辑统计、地点收藏管理、数据重置等较完整功能。本地路线推荐可以根据时间、区域和简化距离给出“时间优先、少换乘、距离优先”三类方案；交通、备忘、账单和收藏夹让作品更接近完整旅行助手。后续可以继续扩展实时路况、公共交通换乘详情、地图真实 polyline、长按拖拽排序和旅行回顾，使作品具有更强的实用价值。
