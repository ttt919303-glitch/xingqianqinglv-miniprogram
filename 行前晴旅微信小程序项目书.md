# 行前晴旅微信小程序项目书

## 一、项目简介

“行前晴旅”是一款面向旅行出行场景的微信小程序，副标题为“轻松规划每一次出发”。作品围绕出发前最常见的准备问题展开，帮助用户查看即将到来的行程、整理随身物品、确认交通时间、安排当天景点，并通过清晰的卡片式界面减少出行前的遗漏和焦虑。

本作品主题积极健康，内容聚焦旅行规划与生活效率提升。界面采用蓝天、白色卡片、暖色插画感元素组成清新明亮的视觉风格，适合课程展示和日常使用场景。

## 二、需求分析

### 1. 用户需求

旅行前用户通常需要同时确认目的地、日期、交通方式、游玩景点和行李物品。若信息分散在聊天记录、备忘录和购票软件中，容易出现遗漏。因此，小程序需要提供一个轻量、直观、便于浏览的行前整理工具。

### 2. 功能需求

- 首页展示即将出发的行程、倒计时、准备进度和旅行小贴士。
- 物品清单按类别展示，包含证件、电子产品、衣物、旅行用品、洗漱用品、药品等。
- 分类详情页支持查看全部、未打包、已打包物品，并能勾选准备状态。
- 行程计划页展示出行日期、交通方式、交通时间、景点安排，并提供“时间优先、少换乘、距离优先”三种路线推荐策略。当前版本已接入腾讯位置服务 WebService，可优先获取真实路线，接口不可用时保留本地推荐方案。
- 模板页提供商务出行、周末短途、毕业旅行、亲子旅行等不同场景参考。
- 我的页面展示作品简介、准备统计和后续扩展方向。
- 数据管理支持清空勾选、恢复初始数据、删除自定义行程和删除自定义物品。

### 3. 非功能需求

- 页面结构清晰，页面数量控制在 4 至 10 页之间。
- 页面深度控制在 2 至 3 层，方便用户快速返回和切换。
- 文字简洁通顺，信息表达明确。
- 设计风格协调统一，色彩搭配清新自然。
- 项目文件命名规范，能在微信开发者工具中正常运行。

## 三、系统设计

### 1. 页面结构

小程序共设计 7 个页面：

- 首页：展示行程卡片、倒计时、准备进度和小贴士。
- 物品清单页：展示各类物品的准备进度。
- 分类详情页：展示单个分类下的物品并支持勾选。
- 行程计划页：展示交通安排、当天景点和推荐路线。
- 模板页：展示不同旅行场景的准备模板。
- 我的页面：展示项目说明和统计信息。
- 新增页面：提供新增行程或物品的演示入口。

### 2. 导航结构

底部导航包含：首页、物品清单、加号、模板、我的。首页和清单页可以继续进入行程计划页和分类详情页，使页面深度保持在合理范围内。

### 3. 数据结构

项目采用本地静态数据作为基础数据，包括行程数据、清单分类数据和模板数据。用户新增的行程、物品、打包状态使用 `wx.setStorageSync` 保存，便于用户切换页面后保持操作结果。系统还提供恢复初始数据功能，方便课堂演示时重新回到默认状态。

## 四、功能实现

### 1. 首页功能

首页展示上海、杭州、苏州三个示例行程。每张行程卡片包含目的地、日期、出发倒计时、准备数量和行程小贴士。用户点击行程卡片可以进入对应的行程计划页。

### 2. 物品清单功能

物品清单页按分类展示准备情况，每个分类显示图标、名称、已准备数量和进度条。用户可以进入分类详情页查看具体物品，也可以使用“一键打包完成”快速完成演示。

### 3. 分类详情功能

分类详情页提供“全部、未打包、已打包”三个筛选选项。物品以卡片网格展示，包含图标、名称、数量和勾选状态。点击物品卡片即可切换是否已准备。

### 4. 行程计划功能

行程计划页展示目的地、日期、交通工具、交通时间、耗时和当天景点安排。系统会根据景点时间、区域、建议停留时间和简化坐标生成本地路线方案，支持“时间优先、少换乘、距离优先”三种策略切换。页面会展示推荐顺序、预计总用时、交通衔接时间、非步行换乘次数和每段交通建议。例如上海行程可根据策略生成“外滩 -> 南京路步行街 -> 豫园 -> 陆家嘴”等路线说明，帮助用户减少路线折返。

在此基础上，项目接入腾讯位置服务 WebService。用户点击“获取腾讯路线”后，小程序会对景点名称进行地理编码，并按每段路线调用腾讯地图路线规划接口，生成真实距离、预计交通时间和路线来源说明。若接口调用失败、网络不可用或小程序后台未配置合法 request 域名，页面会提示原因并继续展示本地路线推荐，保证核心功能可用。

本机开发演示阶段已在项目配置中关闭开发者工具 URL 校验，便于课堂环境下直接调试接口。若进行真机预览、上传体验版或正式发布，仍需要在微信公众平台后台配置 `https://apis.map.qq.com` 为 request 合法域名。

为避免公开仓库泄露接口凭据，腾讯位置服务 Key 使用本地私有配置文件 `config/tencent-map.js` 保存，仓库中仅提交 `config/tencent-map.example.js` 作为填写示例。

### 5. 模板与我的页面

模板页提供多种旅行场景，帮助用户根据出行目的快速建立准备思路。点击模板后会向清单中补充常用物品，并避免重复添加同名物品。我的页面展示应用名称、准备进度、行程数量、清单分类数量和作品说明，同时提供清空勾选与恢复初始数据功能，使作品主题更加完整。

## 五、界面设计

小程序整体采用“蓝天旅行 + 暖色插画 + 白色卡片”的视觉方向。主色为清爽蓝，辅色为薄荷绿和浅橙，背景为浅灰白。页面信息以卡片承载，重点内容使用较大的标题和数字展示，提高浏览效率。

首页使用天空渐变、云朵、热气球、飞机等轻量装饰元素营造旅行氛围。清单页和详情页注重实用性，通过进度条、分类图标和筛选标签帮助用户快速判断准备状态。

## 六、测试说明

测试时主要检查以下内容：

- 微信开发者工具能够正常打开并编译项目。
- 底部导航的五个入口均能正常切换。
- 首页三个行程卡片显示正确，点击后能进入行程计划页。
- 物品清单页能正确统计已准备数量和总数量。
- 分类详情页的“全部、未打包、已打包”筛选功能正确。
- 点击物品后勾选状态能够变化，并在返回页面后保持。
- 分类详情页支持本类全选、本类清空，长按自定义物品可删除。
- 行程计划页能展示交通方式、交通时间、景点安排和推荐路线，切换路线策略后路线摘要和分段建议能同步刷新。
- 新增页可以保存自定义行程和自定义物品，保存后能在对应页面显示。
- 行程路线页点击“获取腾讯路线”后，能优先展示腾讯地图路线；接口不可用时能回退到本地推荐。
- 模板页重复套用时不会重复添加同名物品。
- 我的页面可以清空勾选并恢复初始数据。
- 页面文字没有明显错别字，布局层次清楚。

项目还预留了 `agent-bridge` 自动化联调配置，可配合 `miniprogram-agent-bridge` 在微信开发者工具中执行新增物品、新增行程、路线推荐页等冒烟流程，便于后续检查页面跳转、数据保存和页面 data 报告。

## 七、项目总结

本项目围绕旅行出行准备这一明确主题展开，避免了功能堆砌，重点突出“行前规划”和“物品整理”。通过首页行程卡片、分类清单、景点时间轴和模板场景，小程序形成了较完整的使用闭环。

当前版本主要采用本地数据实现，并接入腾讯位置服务作为增强能力，适合课程作业展示和基础交互演示。项目已经具备行程展示、物品管理、模板套用、本地路线推荐、腾讯地图路线获取、数据重置等较完整功能。本地路线推荐可以根据时间、区域和简化距离给出“时间优先、少换乘、距离优先”三类方案；腾讯地图路线可以进一步提供真实距离和交通耗时。后续可以继续扩展实时路况、公共交通换乘详情和地图可视化路线，使作品具有更强的实用价值。
